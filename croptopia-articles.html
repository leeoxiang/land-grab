Croptopia Plots Are Stakes, Not Listings

Croptopia · 8 min read

The first thing to understand about Croptopia is that a plot is not a listing. It isn't a row in a database with a price next to it. It's a stake — a non-custodial position in the protocol, held open by locked USDC and closed only when you transfer it or walk away.

The world is finite. There are exactly 100 plots, distributed across four tiers, ever. They don't mint. They don't airdrop. They don't get added in the next season. Once they're claimed, they're claimed, and the only way more come back to the market is when somebody decides to abandon theirs.

That scarcity isn't artificial. It's enforced at the program level on Solana.

The plot account

Every plot in Croptopia is a Solana program-derived account, seeded by its plot ID. Once derived, the address is permanent — you can't fork the world, and you can't squat on plots you don't own.

#[account]
pub struct Plot {
    pub owner:         Pubkey,        // Pubkey::default() when unclaimed
    pub tier:          PlotTier,      // Bronze | Silver | Gold | Diamond
    pub locked_amount: u64,           // USDC base units in vault
    pub claimed_at:    i64,           // unix seconds
    pub upgrade_level: u8,            // 1..=4
    pub last_harvest:  i64,
    pub bump:          u8,
}

#[derive(AnchorSerialize, AnchorDeserialize, Clone, Copy, PartialEq)]
pub enum PlotTier {
    Bronze,
    Silver,
    Gold,
    Diamond,
}

Each plot also has its own vault PDA — a token account whose authority is the program itself, derived from the plot seed. When you stake into a plot, the USDC moves from your associated token account into that vault. The vault holds it for as long as you hold the plot.

Tier scarcity, by design

The 100 plots break down by tier:

  • Bronze (50) — base yield, 4 crop slots, 1 animal slot.
  • Silver (30) — 1.5× yield, 6 crop slots, 2 animal slots.
  • Gold (15) — 2× yield, 8 crop slots, 3 animal slots.
  • Diamond (5) — 3× yield, 10 crop slots, 5 animal slots.

Stake costs scale with tier. Bronze is the on-ramp. Diamond is the prize.

impl PlotTier {
    pub const fn claim_cost(&self) -> u64 {
        match self {
            PlotTier::Bronze  => 1_000_000,    //  1 USDC
            PlotTier::Silver  => 5_000_000,    //  5 USDC
            PlotTier::Gold    => 15_000_000,   // 15 USDC
            PlotTier::Diamond => 50_000_000,   // 50 USDC
        }
    }

    pub const fn supply(&self) -> u8 {
        match self {
            PlotTier::Bronze  => 50,
            PlotTier::Silver  => 30,
            PlotTier::Gold    => 15,
            PlotTier::Diamond => 5,
        }
    }
}

The 5-plot cap

The protocol caps wallets at five plots. Not because of opinion. Because a single whale could otherwise drain the diamond supply and leave the rest of the world farming bronze forever.

pub fn claim_plot(ctx: Context<ClaimPlot>) -> Result<()> {
    let plot   = &mut ctx.accounts.plot;
    let player = &mut ctx.accounts.player;

    require!(plot.owner == Pubkey::default(), ErrorCode::AlreadyClaimed);
    require!(player.plot_count < 5,            ErrorCode::TooManyPlots);

    let cost = plot.tier.claim_cost();
    transfer_to_vault(&ctx, cost)?;

    plot.owner         = ctx.accounts.player.key();
    plot.locked_amount = cost;
    plot.claimed_at    = Clock::get()?.unix_timestamp;
    plot.upgrade_level = 1;

    player.plot_count += 1;
    Ok(())
}

Five plots is enough to build a serious farm. It isn't enough to corner the market.

Forfeit on abandon

This is the part most farming games get wrong. They let you walk away clean. Croptopia doesn't.

When you abandon a plot, your stake doesn't return to your wallet. Half is burned. Half goes to the protocol treasury. The plot returns to the world supply, available for the next farmer.

pub fn abandon_plot(ctx: Context<AbandonPlot>) -> Result<()> {
    let plot = &mut ctx.accounts.plot;
    require!(plot.owner == ctx.accounts.player.key(), ErrorCode::NotOwner);

    let stake    = plot.locked_amount;
    let burn     = stake / 2;
    let treasury = stake - burn;

    transfer_from_vault(&ctx, ctx.accounts.burn_vault,     burn)?;
    transfer_from_vault(&ctx, ctx.accounts.treasury_vault, treasury)?;

    plot.owner         = Pubkey::default();
    plot.locked_amount = 0;
    plot.claimed_at    = 0;
    plot.upgrade_level = 1;

    ctx.accounts.player.plot_count -= 1;
    Ok(())
}

This is the line that separates Croptopia from a farm sim. The cost of holding land is real. The cost of giving up on land is also real. You think harder about which plot you want before you stake into it. You think harder about whether to defend a plot you already own.

Land you don't believe in, you don't claim. Land you do, you defend.

What you actually own

Holding a Croptopia plot gets you four things, all on-chain, all tied to the same account:

  1. Yield rights. The plot accrues crops, animal products, and tree harvests on a tier-weighted schedule. Yield credits your wallet automatically when you harvest.
  2. Slot capacity. Bronze fits four crops; Diamond fits ten. Slots translate directly into harvest density.
  3. Upgrade headroom. Plots can be upgraded to level 4 for compounding yield bonuses (more on this in part 3).
  4. Transferability. You can list your plot for any price you want, to any wallet you want. Or to nobody specific, on the open market.

What you don't get is the right to hold the plot for free. The stake is the rent. The rent is the alignment.

Why this matters

Most "play to earn" games confuse activity with commitment. They reward logging in. They reward grinding. They reward being early. None of that filters for whether you actually believe in the world you're farming.

A stake does. A stake says: I am willing to lock real value into this position, and I accept that walking away costs me. That signal is the floor under everything else in Croptopia — the markets, the tribes, the leaderboards, the raids. It's why every system on top of plots feels weighted, instead of frictionless.

Croptopia is not a free-to-play game with optional NFTs. It's a protocol where land is the primitive, and farming it is what you do with the land you've bought.

Two Markets Inside One Game: The Croptopia Exchange

Croptopia · 9 min read

Croptopia runs two markets. One trades the land itself. The other trades what comes off the land. Both are on Solana, both are atomic, and neither has a market maker — players make the market, and the protocol settles it.

Most games either give you a fixed-price NPC vendor or a chat channel where people post WTS messages and pray. Croptopia gives you an order book for crops and a peer-to-peer auction layer for plots. Different machinery for different problems.

Plots are unique. Crops are fungible.

That single distinction drives the entire market design. A plot has a tier, a position in the world, a name, an upgrade level, a history. Two diamond plots aren't interchangeable. So plot trades are negotiated, one offer at a time, with optional targeting.

A bushel of wheat is a bushel of wheat. Quantity matters, source doesn't. So crops trade through a continuous order book that matches the lowest ask against the highest bid the moment a new order lands.

The plot exchange

Every plot offer is a separate on-chain account, seeded by the plot it's attached to. Croptopia enforces one open offer per plot at a time — listing a plot a second time cancels the first.

#[account]
pub struct PlotOffer {
    pub plot:       Pubkey,
    pub seller:     Pubkey,
    pub buyer:      Option<Pubkey>,   // None = open market, Some = targeted
    pub price:      u64,              // USDC base units
    pub status:     OfferStatus,      // Open | Accepted | Cancelled
    pub created_at: i64,
    pub bump:       u8,
}

#[derive(AnchorSerialize, AnchorDeserialize, Clone, Copy, PartialEq)]
pub enum OfferStatus { Open, Accepted, Cancelled }

The interesting field is buyer: Option<Pubkey>. Set it to None and the offer is public — anyone with USDC and a free plot slot can take it. Set it to Some(pubkey) and the offer is targeted — only that wallet can accept. This is how plot OTC works. You DM someone, agree on a number, list to their wallet specifically. They accept. Settlement is atomic.

Atomic settlement

The accept instruction is where most marketplaces would get this wrong. Between the moment an offer is listed and the moment a buyer accepts, the world keeps moving. The seller might list elsewhere. The plot might change hands through a raid. The seller might cancel.

Croptopia's accept is guarded against all three with a single optimistic concurrency check.

pub fn accept_plot_offer(ctx: Context<AcceptPlotOffer>) -> Result<()> {
    let offer = &mut ctx.accounts.offer;
    let plot  = &mut ctx.accounts.plot;
    let buyer = ctx.accounts.buyer.key();

    require!(offer.status == OfferStatus::Open,        ErrorCode::OfferClosed);
    require!(plot.owner   == offer.seller,             ErrorCode::SellerNoLongerOwns);
    require!(offer.seller != buyer,                    ErrorCode::SelfTrade);
    require!(
        offer.buyer.map_or(true, |b| b == buyer),
        ErrorCode::OfferNotForYou,
    );

    // Protocol fee: 2.5% of price
    let fee = offer.price * 25 / 1000;
    let net = offer.price - fee;

    // USDC: buyer ATA -> seller ATA + treasury
    transfer_buyer_to(&ctx, ctx.accounts.seller_token,   net)?;
    transfer_buyer_to(&ctx, ctx.accounts.treasury_token, fee)?;

    // Plot ownership transfer (atomic with payment)
    plot.owner      = buyer;
    plot.claimed_at = Clock::get()?.unix_timestamp;

    offer.status = OfferStatus::Accepted;
    offer.buyer  = Some(buyer);
    Ok(())
}

The line that does the heavy lifting is require!(plot.owner == offer.seller). If anything changed the plot's owner since the offer was posted — another sale, an abandonment, a raid that flipped the deed — that check fails, the transaction reverts, and no USDC moves. The buyer's funds are safe. The seller's plot is safe.

This is what people mean when they say "atomic." Either everything happens or nothing happens. There's no half-state. There's no race.

The crop order book

The other market is the crop exchange. This is where wheat, eggs, milk, fish, honey, magic herbs — everything that comes off a plot — gets traded between players for USDC.

#[account]
pub struct Order {
    pub maker:      Pubkey,
    pub item:       ItemType,        // Wheat | Corn | Egg | Milk | ...
    pub side:       Side,            // Bid | Ask
    pub quantity:   u64,
    pub remaining:  u64,
    pub price:      u64,             // USDC per unit
    pub status:     OrderStatus,
    pub created_at: i64,
}

#[derive(AnchorSerialize, AnchorDeserialize, Clone, Copy)]
pub enum Side         { Bid, Ask }
#[derive(AnchorSerialize, AnchorDeserialize, Clone, Copy)]
pub enum OrderStatus  { Open, Filled, Cancelled }

When you place a sell order for 100 wheat at 2 USDC each, the program does three things: it locks 100 wheat from your inventory into an item escrow account, it inserts the order, and it runs the matching engine.

pub fn place_order(ctx: Context<PlaceOrder>, args: OrderArgs) -> Result<()> {
    let order = &mut ctx.accounts.order;

    match args.side {
        Side::Ask => {
            // Reserve inventory: items move into program escrow
            burn_inventory(&ctx, args.item, args.quantity)?;
            credit_escrow(&ctx, args.item, args.quantity)?;
        }
        Side::Bid => {
            // Reserve USDC at limit price
            transfer_to_escrow(&ctx, args.quantity * args.price)?;
        }
    }

    order.maker     = ctx.accounts.maker.key();
    order.item      = args.item;
    order.side      = args.side;
    order.quantity  = args.quantity;
    order.remaining = args.quantity;
    order.price     = args.price;
    order.status    = OrderStatus::Open;

    match_book(&ctx, args.item)?;
    Ok(())
}

Matching

The matching engine is straightforward in shape and ruthless in execution. Walk the asks lowest-first. Walk the bids highest-first. While the best bid is at or above the best ask, fill at the ask price (the patient seller wins ties). Stop when the books cross or one side runs out.

pub fn match_book(ctx: &Context<PlaceOrder>, item: ItemType) -> Result<()> {
    let mut asks = best_asks(item, 10)?;
    let mut bids = best_bids(item, 10)?;

    for ask in asks.iter_mut() {
        for bid in bids.iter_mut() {
            if bid.price < ask.price        { break; }
            if bid.maker == ask.maker        { continue; }   // self-match guard

            let qty   = ask.remaining.min(bid.remaining);
            let price = ask.price;
            let gross = qty * price;
            let fee   = gross * 25 / 1000;
            let net   = gross - fee;

            settle_payment(ctx, ask.maker, net, fee)?;       // bid escrow -> ask maker
            settle_delivery(ctx, bid.maker, item, qty)?;     // ask escrow -> bid maker

            ask.remaining -= qty;
            bid.remaining -= qty;
            if ask.remaining == 0 { ask.status = OrderStatus::Filled; }
            if bid.remaining == 0 { bid.status = OrderStatus::Filled; }
            if ask.remaining == 0 { break; }
        }
    }
    Ok(())
}

The 2.5% fee is split: most of it goes to the protocol treasury, a smaller portion is burned. The self-match guard stops a wallet from arbing its own orders against itself to fake volume.

What this builds

The two markets work because they don't try to do each other's jobs. Plot trades are slow and negotiated and one-of-a-kind, so they're peer-to-peer auctions with optional targeting. Crop trades are fast and fungible and continuous, so they're a real order book that clears every time someone shows up.

Together they make the world feel commercial. Wheat doesn't have a fixed price. It has the price the highest current bid wants to pay. Plots don't have a tier sticker price. They have whatever someone's willing to part with theirs for. And the protocol gets out of the way once the rules are set.

We didn't build a marketplace. We built a primitive. The players make the market.

Tribes, Alliances, and Yield Compounding

Croptopia · 9 min read

The interesting question in Croptopia isn't whether your wheat is ready. It's how much wheat you actually receive when you harvest it.

The base number printed on a crop is a starting point. Every crop, every animal product, every tree harvest gets passed through a stack of multipliers before it lands in your inventory. Plot upgrades. Tribe membership. Alliance pacts. Time-of-day bonuses. They compose. They compound. And the gap between a casual harvester and a maxed-out one is multiplicative, not additive.

The base yield

Every plant in Croptopia has a base yield and a sell value. Wheat returns 1 wheat per cycle and trades for around 2 USDC. Magic herbs return less often but trade for fifty times as much. The base is the number you'd get if you owned a level-1 plot, joined no group, harvested at midnight on a slow day, and walked away.

Almost nobody plays Croptopia that way.

Plot upgrades

Plots upgrade through four levels. Each level multiplies the yield of everything on the plot — crops, animals, trees, fish, honey, all of it.

  • Level 1 — base yield (1.00×). Free with claim.
  • Level 2 — 1.25× yield. Costs 5 USDC.
  • Level 3 — 1.5× yield. Costs 15 USDC.
  • Level 4 — 2.0× yield. Costs 40 USDC.

Upgrade fees go to the treasury vault, with a portion burned. The bonus is permanent for as long as you own the plot, which means an upgrade pays itself back in harvests as soon as your harvest cadence catches up.

Tribes

Tribes are the heavy social structure in Croptopia. Up to ten members. A leader. An invite code. Applications you accept or reject. A kick instruction the leader can sign. A name and a tag the world sees on the leaderboard.

#[account]
pub struct Tribe {
    pub id:           u64,
    pub name:         [u8; 32],
    pub tag:          [u8; 4],
    pub leader:       Pubkey,
    pub home_plot:    Pubkey,
    pub member_count: u8,        // 1..=10
    pub invite_code:  [u8; 8],
    pub created_at:   i64,
    pub bump:         u8,
}

#[account]
pub struct TribeMember {
    pub tribe:    Pubkey,
    pub wallet:   Pubkey,
    pub joined:   i64,
    pub is_leader: bool,
    pub bump:     u8,
}

The tribe yield bonus splits leader and member rewards asymmetrically:

  • Members get a flat +10% on every harvest.
  • Leaders get +5% per member, scaling with the size of their tribe. A full ten-member tribe earns its leader a +45% bonus on every harvest (5% × the nine non-leader members).

The asymmetry is the whole point. Leaders carry recruiting and moderation costs. Members carry consistency. The bonuses match the labor.

Alliances

Alliances are the lightweight structure. Up to five members. No leader. No applications, no kicks, just join and leave. Everyone gets a flat +5% bonus on harvests for as long as they're in the alliance.

#[account]
pub struct Alliance {
    pub id:           u64,
    pub name:         [u8; 32],
    pub member_count: u8,        // 1..=5
    pub created_at:   i64,
    pub bump:         u8,
}

Tribes are guilds. Alliances are pacts. A player can be in one tribe and one alliance at the same time, and the bonuses stack — because they're rewarding two different kinds of social commitment.

Golden Hour

Every six hours, for one hour, the protocol enters Golden Hour. Every harvest in the world gets a flat +20% bonus during the window. There's no cron job, no oracle, no off-chain trigger — it's a deterministic schedule on the program clock.

pub fn is_golden_hour(now: i64) -> bool {
    const CYCLE_SECONDS:   i64 = 6 * 60 * 60;   // 6 hours
    const WINDOW_SECONDS:  i64 = 1 * 60 * 60;   // 1 hour
    let phase = now.rem_euclid(CYCLE_SECONDS);
    phase < WINDOW_SECONDS
}

It's the same window worldwide. Players coordinate around it. Tribe leaders organize harvest rallies. The market sees a volume spike. Good farmers schedule themselves.

The compounding stack

Here is what every harvest in Croptopia actually computes.

pub fn compute_yield(
    base:           u64,
    upgrade_level:  u8,
    in_tribe:       bool,
    is_tribe_leader: bool,
    tribe_members:  u8,
    in_alliance:    bool,
    golden_hour:    bool,
) -> u64 {
    // Multipliers expressed in basis points (10_000 = 1.00x)
    let upgrade_bps = match upgrade_level {
        1 => 10_000,
        2 => 12_500,
        3 => 15_000,
        4 => 20_000,
        _ => 10_000,
    };

    let tribe_bps = if !in_tribe {
        10_000
    } else if is_tribe_leader {
        10_000 + 500 * (tribe_members.saturating_sub(1) as u32) as u64
    } else {
        11_000  // +10%
    };

    let alliance_bps = if in_alliance { 10_500 } else { 10_000 };
    let golden_bps   = if golden_hour { 12_000 } else { 10_000 };

    let mut yield_amt = base;
    yield_amt = yield_amt * upgrade_bps  / 10_000;
    yield_amt = yield_amt * tribe_bps    / 10_000;
    yield_amt = yield_amt * alliance_bps / 10_000;
    yield_amt = yield_amt * golden_bps   / 10_000;
    yield_amt
}

Multipliers compose multiplicatively, not additively. That single design choice is what makes Croptopia reward dedication out of proportion to effort.

What four bonuses look like

Take a player on a level-4 diamond plot, member of a 10-person tribe (not the leader), in a 5-person alliance, harvesting during Golden Hour. Their wheat plot yields a base of 100 wheat per cycle.

let base     = 100;                    // base yield
let upgrade  = base * 200 / 100;       // 200    (level 4: 2.0x)
let tribe    = upgrade * 110 / 100;    // 220    (member: +10%)
let alliance = tribe   * 105 / 100;    // 231    (alliance: +5%)
let golden   = alliance * 120 / 100;   // 277    (golden hour: +20%)
//
// Final harvest: 277 wheat from a base of 100.

That's 2.77× what an unaffiliated, level-1, off-hours farmer would receive from the same plot at the same moment. Not because we boosted them. Because they did the work to earn every multiplier in the stack.

And that's just the harvest itself. The wheat then sells through the order book to other players. The order book is liquid because the world keeps farming. The world keeps farming because the yield rewards farming. The yield rewards farming because everyone is locked into a stake (see part one). The stake is real because abandoning it costs you (also part one). The market is real because the trades settle atomically (part two).

This is the loop. Plots are the primitive. Tribes and alliances are the accelerator. The order book is the exit. Yield is the reason any of it matters.

Anyone can show up. Fewer can build the structure. The protocol pays the difference.
