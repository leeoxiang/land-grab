Wheat World Plots Are Stakes, Not Listings

Wheat World · 8 min read

The first thing to understand about Wheat World is that a plot is not a listing. It isn't a row in a database with a price next to it. It's a stake — a non-custodial position in the protocol, held open by locked USDC and closed only when you transfer it or walk away.

The world is finite. There are exactly 100 plots, distributed across four tiers, ever. They don't mint. They don't airdrop. They don't get added in the next season. Once they're claimed, they're claimed, and the only way more come back to the market is when somebody decides to abandon theirs.

That scarcity isn't artificial. It's enforced at the contract level on Base.

The plot record

Every plot in Wheat World lives in a single mapping inside the protocol contract on Base. The plot ID is its key. Once written, the slot is permanent — you can't fork the world, and you can't squat on plots you don't own.

// contracts/WheatWorld.sol
struct Plot {
    address owner;          // address(0) when unclaimed
    PlotTier tier;          // Bronze | Silver | Gold | Diamond
    uint128  lockedAmount;  // USDC (6 decimals) held by the contract
    uint64   claimedAt;
    uint8    upgradeLevel;  // 1..=4
    uint64   lastHarvest;
}

enum PlotTier { Bronze, Silver, Gold, Diamond }

mapping(uint8 => Plot) public plots;

Locked USDC sits inside the contract itself — there is no per-plot wallet, no external custodian. The contract's USDC balance is the protocol's collateral pool. Plot ownership is the right to withdraw your stake under specific conditions, and the right to harvest from the land while you hold it.

Tier scarcity, by design

The 100 plots break down by tier:

  • Bronze (50) — base yield, 4 crop slots, 1 animal slot.
  • Silver (30) — 1.5× yield, 6 crop slots, 2 animal slots.
  • Gold (15) — 2× yield, 8 crop slots, 3 animal slots.
  • Diamond (5) — 3× yield, 10 crop slots, 5 animal slots.

Stake costs scale with tier. Bronze is the on-ramp. Diamond is the prize.

function claimCost(PlotTier tier) internal pure returns (uint128) {
    if (tier == PlotTier.Bronze)  return  1e6;   //  1 USDC
    if (tier == PlotTier.Silver)  return  5e6;   //  5 USDC
    if (tier == PlotTier.Gold)    return 15e6;   // 15 USDC
    if (tier == PlotTier.Diamond) return 50e6;   // 50 USDC
    revert InvalidTier();
}

The 5-plot cap

The protocol caps wallets at five plots. Not because of opinion. Because a single whale could otherwise drain the diamond supply and leave the rest of the world farming bronze forever.

function claimPlot(uint8 plotId, PlotTier tier) external {
    Plot storage p = plots[plotId];
    if (p.owner != address(0))            revert AlreadyClaimed();
    if (plotsOwned[msg.sender] >= 5)      revert TooManyPlots();

    uint128 cost = claimCost(tier);
    USDC.transferFrom(msg.sender, address(this), cost);

    p.owner         = msg.sender;
    p.tier          = tier;
    p.lockedAmount  = cost;
    p.claimedAt     = uint64(block.timestamp);
    p.upgradeLevel  = 1;

    plotsOwned[msg.sender] += 1;
    emit PlotClaimed(plotId, msg.sender, tier, cost);
}

Five plots is enough to build a serious farm. It isn't enough to corner the market.

Forfeit on abandon

This is the part most farming games get wrong. They let you walk away clean. Wheat World doesn't.

When you abandon a plot, your stake doesn't return to your wallet. Half is burned. Half goes to the protocol treasury. The plot returns to the world supply, available for the next farmer.

function abandonPlot(uint8 plotId) external {
    Plot storage p = plots[plotId];
    if (p.owner != msg.sender) revert NotOwner();

    uint128 stake    = p.lockedAmount;
    uint128 burn     = stake / 2;
    uint128 treasury = stake - burn;

    USDC.transfer(BURN_ADDRESS,    burn);
    USDC.transfer(TREASURY,        treasury);

    p.owner         = address(0);
    p.lockedAmount  = 0;
    p.claimedAt     = 0;
    p.upgradeLevel  = 1;

    plotsOwned[msg.sender] -= 1;
    emit PlotAbandoned(plotId, stake);
}

This is the line that separates Wheat World from a farm sim. The cost of holding land is real. The cost of giving up on land is also real. You think harder about which plot you want before you stake into it. You think harder about whether to defend a plot you already own.

Land you don't believe in, you don't claim. Land you do, you defend.

What you actually own

Holding a Wheat World plot gets you four things, all on-chain, all tied to the same record:

  1. Yield rights. The plot accrues crops, animal products, and tree harvests on a tier-weighted schedule. Yield credits your wallet automatically when you harvest.
  2. Slot capacity. Bronze fits four crops; Diamond fits ten. Slots translate directly into harvest density.
  3. Upgrade headroom. Plots can be upgraded to level 4 for compounding yield bonuses (more on this in part 3).
  4. Transferability. You can list your plot for any price you want, to any wallet you want. Or to nobody specific, on the open market.

What you don't get is the right to hold the plot for free. The stake is the rent. The rent is the alignment.

Why this matters

Most "play to earn" games confuse activity with commitment. They reward logging in. They reward grinding. They reward being early. None of that filters for whether you actually believe in the world you're farming.

A stake does. A stake says: I am willing to lock real value into this position, and I accept that walking away costs me. That signal is the floor under everything else in Wheat World — the markets, the tribes, the leaderboards, the raids. It's why every system on top of plots feels weighted, instead of frictionless.

Wheat World is not a free-to-play game with optional NFTs. It's a protocol where land is the primitive, and farming it is what you do with the land you've bought.

Two Markets Inside One Game: The Wheat World Exchange

Wheat World · 9 min read

Wheat World runs two markets. One trades the land itself. The other trades what comes off the land. Both are on Base, both are atomic, and neither has a market maker — players make the market, and the contract settles it.

Most games either give you a fixed-price NPC vendor or a chat channel where people post WTS messages and pray. Wheat World gives you an order book for crops and a peer-to-peer auction layer for plots. Different machinery for different problems.

Plots are unique. Crops are fungible.

That single distinction drives the entire market design. A plot has a tier, a position in the world, a name, an upgrade level, a history. Two diamond plots aren't interchangeable. So plot trades are negotiated, one offer at a time, with optional targeting.

A bushel of wheat is a bushel of wheat. Quantity matters, source doesn't. So crops trade through a continuous order book that matches the lowest ask against the highest bid the moment a new order lands.

The plot exchange

Every plot offer is a separate record in a mapping keyed by plot ID. Wheat World enforces one open offer per plot at a time — listing a plot a second time cancels the first.

struct PlotOffer {
    address seller;
    address buyer;        // address(0) = open market, non-zero = targeted
    uint128 price;        // USDC base units
    OfferStatus status;   // Open | Accepted | Cancelled
    uint64  createdAt;
}

enum OfferStatus { Open, Accepted, Cancelled }

mapping(uint8 => PlotOffer) public offers;

The interesting field is address buyer. Set it to address(0) and the offer is public — anyone with USDC and a free plot slot can take it. Set it to a specific address and the offer is targeted — only that wallet can accept. This is how plot OTC works. You DM someone, agree on a number, list to their wallet specifically. They accept. Settlement is atomic.

Atomic settlement

The accept function is where most marketplaces would get this wrong. Between the moment an offer is listed and the moment a buyer accepts, the world keeps moving. The seller might list elsewhere. The plot might change hands through a raid. The seller might cancel.

Wheat World's accept is guarded against all three with a single optimistic concurrency check.

function acceptPlotOffer(uint8 plotId) external {
    PlotOffer storage o = offers[plotId];
    Plot      storage p = plots[plotId];
    address           buyer = msg.sender;

    if (o.status != OfferStatus.Open)                  revert OfferClosed();
    if (p.owner  != o.seller)                          revert SellerNoLongerOwns();
    if (o.seller == buyer)                             revert SelfTrade();
    if (o.buyer  != address(0) && o.buyer != buyer)    revert OfferNotForYou();

    // Protocol fee: 2.5%
    uint128 fee = uint128(uint256(o.price) * 25 / 1000);
    uint128 net = o.price - fee;

    // USDC: buyer -> seller (net) + treasury (fee)
    USDC.transferFrom(buyer, o.seller, net);
    USDC.transferFrom(buyer, TREASURY, fee);

    // Plot ownership transfer (atomic with payment)
    p.owner      = buyer;
    p.claimedAt  = uint64(block.timestamp);

    o.status = OfferStatus.Accepted;
    o.buyer  = buyer;

    emit PlotTraded(plotId, o.seller, buyer, o.price);
}

The line that does the heavy lifting is if (p.owner != o.seller). If anything changed the plot's owner since the offer was posted — another sale, an abandonment, a raid that flipped the deed — that check reverts, the transaction unwinds, and no USDC moves. The buyer's funds are safe. The seller's plot is safe.

This is what people mean when they say "atomic." Either everything happens or nothing happens. There's no half-state. There's no race.

The crop order book

The other market is the crop exchange. This is where wheat, eggs, milk, fish, honey, magic herbs — everything that comes off a plot — gets traded between players for USDC.

struct Order {
    address  maker;
    ItemType item;       // Wheat | Corn | Egg | Milk | ...
    Side     side;       // Bid | Ask
    uint128  quantity;
    uint128  remaining;
    uint128  price;      // USDC per unit
    OrderStatus status;
    uint64   createdAt;
}

enum Side        { Bid, Ask }
enum OrderStatus { Open, Filled, Cancelled }

mapping(uint256 => Order) public orders;
uint256 public nextOrderId;

When you place a sell order for 100 wheat at 2 USDC each, the contract does three things: it locks 100 wheat from your inventory into the contract's item ledger, it inserts the order, and it runs the matching engine.

function placeOrder(OrderArgs calldata a) external returns (uint256 id) {
    if (a.quantity == 0 || a.price == 0) revert InvalidOrder();

    id = ++nextOrderId;
    Order storage o = orders[id];
    o.maker     = msg.sender;
    o.item      = a.item;
    o.side      = a.side;
    o.quantity  = a.quantity;
    o.remaining = a.quantity;
    o.price     = a.price;
    o.status    = OrderStatus.Open;
    o.createdAt = uint64(block.timestamp);

    if (a.side == Side.Ask) {
        _lockItems(msg.sender, a.item, a.quantity);          // inventory escrow
    } else {
        uint128 lock = a.quantity * a.price;
        USDC.transferFrom(msg.sender, address(this), lock);  // USDC escrow
    }

    _matchBook(a.item);
}

Matching

The matching engine is straightforward in shape and ruthless in execution. Walk the asks lowest-first. Walk the bids highest-first. While the best bid is at or above the best ask, fill at the ask price (the patient seller wins ties). Stop when the books cross or one side runs out.

function _matchBook(ItemType item) internal {
    uint256[] memory asks = bestAsks(item, 10);
    uint256[] memory bids = bestBids(item, 10);

    for (uint i; i < asks.length; ++i) {
        Order storage a = orders[asks[i]];
        for (uint j; j < bids.length; ++j) {
            Order storage b = orders[bids[j]];

            if (b.price < a.price)   break;
            if (b.maker == a.maker)  continue;       // self-match guard

            uint128 qty   = a.remaining < b.remaining ? a.remaining : b.remaining;
            uint128 price = a.price;
            uint128 gross = qty * price;
            uint128 fee   = gross * 25 / 1000;
            uint128 net   = gross - fee;

            _settle(a.maker, b.maker, item, qty, net, fee);

            a.remaining -= qty;
            b.remaining -= qty;
            if (a.remaining == 0) { a.status = OrderStatus.Filled; }
            if (b.remaining == 0) { b.status = OrderStatus.Filled; }

            emit OrderFilled(asks[i], bids[j], qty, price, fee, net);
            if (a.remaining == 0) break;
        }
    }
}

The 2.5% fee is split: most of it goes to the protocol treasury, a smaller portion is burned. The self-match guard stops a wallet from arbing its own orders against itself to fake volume.

What this builds

The two markets work because they don't try to do each other's jobs. Plot trades are slow and negotiated and one-of-a-kind, so they're peer-to-peer auctions with optional targeting. Crop trades are fast and fungible and continuous, so they're a real order book that clears every time someone shows up.

Together they make the world feel commercial. Wheat doesn't have a fixed price. It has the price the highest current bid wants to pay. Plots don't have a tier sticker price. They have whatever someone's willing to part with theirs for. And the contract gets out of the way once the rules are set.

We didn't build a marketplace. We built a primitive. The players make the market.

Tribes, Alliances, and Yield Compounding

Wheat World · 9 min read

The interesting question in Wheat World isn't whether your wheat is ready. It's how much wheat you actually receive when you harvest it.

The base number printed on a crop is a starting point. Every crop, every animal product, every tree harvest gets passed through a stack of multipliers before it lands in your inventory. Plot upgrades. Tribe membership. Alliance pacts. Time-of-day bonuses. They compose. They compound. And the gap between a casual harvester and a maxed-out one is multiplicative, not additive.

The base yield

Every plant in Wheat World has a base yield and a sell value. Wheat returns 1 wheat per cycle and trades for around 2 USDC. Magic herbs return less often but trade for fifty times as much. The base is the number you'd get if you owned a level-1 plot, joined no group, harvested at midnight on a slow day, and walked away.

Almost nobody plays Wheat World that way.

Plot upgrades

Plots upgrade through four levels. Each level multiplies the yield of everything on the plot — crops, animals, trees, fish, honey, all of it.

  • Level 1 — base yield (1.00×). Free with claim.
  • Level 2 — 1.25× yield. Costs 5 USDC.
  • Level 3 — 1.5× yield. Costs 15 USDC.
  • Level 4 — 2.0× yield. Costs 40 USDC.

Upgrade fees go to the treasury, with a portion burned. The bonus is permanent for as long as you own the plot, which means an upgrade pays itself back in harvests as soon as your harvest cadence catches up.

Tribes

Tribes are the heavy social structure in Wheat World. Up to ten members. A leader. An invite code. Applications you accept or reject. A kick function the leader can call. A name and a tag the world sees on the leaderboard.

struct Tribe {
    bytes32 name;
    bytes4  tag;
    address leader;
    uint8   memberCount;     // 1..=10
    bytes8  inviteCode;
    uint64  createdAt;
}

struct TribeMember {
    uint256 tribeId;
    address wallet;
    uint64  joinedAt;
    bool    isLeader;
}

mapping(uint256 => Tribe) public tribes;
mapping(uint256 => mapping(address => TribeMember)) public tribeMembers;

The tribe yield bonus splits leader and member rewards asymmetrically:

  • Members get a flat +10% on every harvest.
  • Leaders get +5% per member, scaling with the size of their tribe. A full ten-member tribe earns its leader a +45% bonus on every harvest (5% × the nine non-leader members).

The asymmetry is the whole point. Leaders carry recruiting and moderation costs. Members carry consistency. The bonuses match the labor.

Alliances

Alliances are the lightweight structure. Up to five members. No leader. No applications, no kicks, just join and leave. Everyone gets a flat +5% bonus on harvests for as long as they're in the alliance.

struct Alliance {
    bytes32 name;
    uint8   memberCount;     // 1..=5
    uint64  createdAt;
}

mapping(uint256 => Alliance) public alliances;

Tribes are guilds. Alliances are pacts. A player can be in one tribe and one alliance at the same time, and the bonuses stack — because they're rewarding two different kinds of social commitment.

Golden Hour

Every six hours, for one hour, the protocol enters Golden Hour. Every harvest in the world gets a flat +20% bonus during the window. There's no cron job, no oracle, no off-chain trigger — it's a deterministic function of block.timestamp.

function isGoldenHour() public view returns (bool) {
    uint256 cycle  = 6 hours;
    uint256 window = 1 hours;
    return (block.timestamp % cycle) < window;
}

It's the same window worldwide. Players coordinate around it. Tribe leaders organize harvest rallies. The market sees a volume spike. Good farmers schedule themselves.

The compounding stack

Here is what every harvest in Wheat World actually computes.

function computeYield(
    uint256 base,
    PlotTier tier,
    uint8   upgradeLevel,
    bool    inTribe,
    bool    isTribeLeader,
    uint8   tribeMembers,
    bool    inAlliance,
    bool    goldenHour
) public pure returns (uint256) {
    // Multipliers expressed in basis points (10_000 = 1.00x)
    uint256 tierBps     = _tierBps(tier);
    uint256 upgradeBps  = _upgradeBps(upgradeLevel);

    uint256 tribeBps = 10_000;
    if (inTribe) {
        if (isTribeLeader) {
            tribeBps = 10_000 + 500 * (uint256(tribeMembers) - 1);
        } else {
            tribeBps = 11_000;  // member: +10%
        }
    }

    uint256 allianceBps = inAlliance ? 10_500 : 10_000;
    uint256 goldenBps   = goldenHour ? 12_000 : 10_000;

    uint256 y = base;
    y = (y * tierBps)     / 10_000;
    y = (y * upgradeBps)  / 10_000;
    y = (y * tribeBps)    / 10_000;
    y = (y * allianceBps) / 10_000;
    y = (y * goldenBps)   / 10_000;
    return y;
}

Multipliers compose multiplicatively, not additively. That single design choice is what makes Wheat World reward dedication out of proportion to effort.

What four bonuses look like

Take a player on a level-4 diamond plot, member of a 10-person tribe (not the leader), in a 5-person alliance, harvesting during Golden Hour. Their wheat plot yields a base of 100 wheat per cycle.

uint256 base     = 100;
uint256 tier     = base     * 30_000 / 10_000;   // 300  (Diamond: 3.0x)
uint256 upgrade  = tier     * 20_000 / 10_000;   // 600  (level 4: 2.0x)
uint256 tribe    = upgrade  * 11_000 / 10_000;   // 660  (member: +10%)
uint256 alliance = tribe    * 10_500 / 10_000;   // 693  (alliance: +5%)
uint256 golden   = alliance * 12_000 / 10_000;   // 831  (golden hour: +20%)
//
// Final harvest: 831 wheat from a base of 100. ~8.31× a casual farmer.

That's 8.31× what an unaffiliated, level-1, off-hours, Bronze-plot farmer would receive from the same harvest tick. Not because we boosted them. Because they did the work to earn every multiplier in the stack.

And that's just the harvest itself. The wheat then sells through the order book to other players. The order book is liquid because the world keeps farming. The world keeps farming because the yield rewards farming. The yield rewards farming because everyone is locked into a stake (see part one). The stake is real because abandoning it costs you (also part one). The market is real because the trades settle atomically (part two).

This is the loop. Plots are the primitive. Tribes and alliances are the accelerator. The order book is the exit. Yield is the reason any of it matters.

Anyone can show up. Fewer can build the structure. The protocol pays the difference.
